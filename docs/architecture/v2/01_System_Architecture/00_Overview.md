# 00 Overview

> 本文档定义 AStock V2 的总体架构视角。
>
> V2 的核心目标是从“规则驱动的量化看板”，升级为“证据驱动的产业知识图谱 + 事件推理系统”。

---

## 1. 背景

AStock V1 已经实现了行情数据、新闻舆情、公告聚合、事件映射、趋势评分、Web 展示等能力。

这些能力证明系统已经可以完成基础的盘后选股和事件观察。

但随着事件类型增多、产业链节点增多、股票映射规则增多，单纯依靠 YAML 或硬编码规则会逐渐出现三个问题：

1. 规则数量快速膨胀。
2. 规则之间缺少统一关系结构。
3. 无法解释一条股票映射关系来自哪里、是否有效。

因此 V2 需要引入更稳定的中间层：Knowledge Graph。

---

## 2. V2 设计目标

AStock V2 的目标不是替代 V1，而是在 V1 之上增加以下能力：

```text
证据沉淀
  ↓
知识图谱
  ↓
事件推理
  ↓
股票暴露度
  ↓
收益验证
  ↓
可解释展示
```

V2 的核心输出不再只是股票列表，而是：

```text
事件影响路径 + 股票暴露度 + 证据来源 + 历史验证结果
```

---

## 3. 设计原则

### 3.1 Evidence First

任何知识图谱关系都应该尽可能绑定证据来源。

例如：

```text
中船特气 -> produces -> 六氟化钨
```

这条关系应该能追溯到：

- 年报
- 招股书
- 官网产品目录
- 公告
- 互动易回复
- 新闻来源

没有证据的关系只能进入候选层，不能直接作为高置信度关系参与核心推理。

---

### 3.2 LLM as Extractor, Not Decision Maker

大模型不负责直接推荐股票。

大模型负责：

- 抽取事件。
- 抽取产品。
- 抽取原材料。
- 抽取上下游。
- 生成候选关系。
- 生成解释文本。

最终决策仍由：

```text
Knowledge Graph + Reasoning Engine + Market Validation
```

共同完成。

---

### 3.3 Graph as Long-Term Memory

YAML 适合表达少量确定规则，但不适合表达复杂、多跳、多来源、可验证的产业关系。

V2 使用知识图谱作为长期记忆，沉淀：

- 公司和产品关系。
- 产品和材料关系。
- 材料和商品关系。
- 商品和行业关系。
- 行业和概念关系。
- 事件和产业节点关系。

---

### 3.4 Validation as Feedback

事件推理不能只停留在“看起来有关”。

系统必须记录事件发生后的真实市场表现：

- T+1 收益。
- T+3 收益。
- T+5 收益。
- T+10 收益。
- 相对行业超额收益。
- 相对指数超额收益。

这些结果用于反向校准事件权重和关系权重。

---

## 4. 总体架构

```text
Data Sources
  ↓
Evidence Collector
  ↓
LLM Extraction / Rule Extraction
  ↓
Candidate Knowledge
  ↓
Knowledge Graph
  ↓
Reasoning Engine
  ↓
Exposure Engine
  ↓
Scoring Engine
  ↓
Validation Engine
  ↓
API Layer
  ↓
Web Dashboard
```

---

## 5. V2 与 V1 的关系

V2 不推翻 V1。

V1 中已有的行情、新闻、公告、信号、前端页面继续保留。

V2 新增的是：

1. 证据层。
2. 图谱层。
3. 推理层。
4. 验证层。
5. 可解释层。

V1 的事件 YAML 可以作为图谱初始化种子。

V1 的股票评分可以继续作为 V2 Scoring Engine 的一个子因子。

---

## 6. 架构边界

### 6.1 离线系统负责

- 拉取行情。
- 拉取公告。
- 拉取新闻。
- 解析年报和招股书。
- 执行 LLM 抽取。
- 构建或更新图谱。
- 执行批量事件推理。
- 计算历史验证结果。

### 6.2 在线系统负责

- 查询图谱。
- 查询事件结果。
- 查询个股解释。
- 展示推理路径。
- 展示验证结果。
- 提供 API 给前端。

---

## 7. 核心交互示例

输入：

```text
市场传出六氟化钨供应紧张。
```

系统处理：

```text
事件抽取：supply_shortage
影响对象：六氟化钨
图谱推理：六氟化钨 -> 电子特气 -> 半导体材料 -> 中船特气
暴露度计算：0.82
行情验证：历史同类事件 T+3 平均超额收益 +X%
前端解释：展示完整路径和证据来源
```

---

## 8. 结论

AStock V2 的总体架构可以概括为：

```text
LLM 负责理解和抽取
Knowledge Graph 负责记忆和推理
Market Data 负责验证和校准
Web 负责解释和展示
```

V2 的核心竞争力不是更多技术指标，而是能够持续沉淀和验证产业链事件知识。

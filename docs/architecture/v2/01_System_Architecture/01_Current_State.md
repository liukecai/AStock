# 01 Current State

> 本文档描述 AStock V1 当前能力、架构问题与 V2 升级动因。

---

## 1. 当前系统定位

AStock V1 已经从脚本演进为可运行的量化研究平台。

当前系统具备：

- A股行情采集。
- 趋势因子计算。
- 新闻舆情聚合。
- 巨潮公告接入。
- RSS 新闻接入。
- 事件与商品映射。
- 股票信号生成。
- Web 页面展示。
- Docker 部署能力。

这说明 V1 已经完成 MVP 阶段目标。

---

## 2. V1 已有能力

### 2.1 数据采集

当前系统已经具备多来源数据采集能力：

```text
AKShare
RSS
巨潮公告
新闻源
```

这些数据源能够支持基础行情、舆情和公告分析。

### 2.2 趋势分析

V1 已经支持基于日线行情的趋势判断，例如：

- 均线结构。
- 价格斜率。
- 趋势评分。
- 成交量变化。

这部分可以继续作为 V2 Scoring Engine 的技术面子因子。

### 2.3 舆情与公告

V1 已经开始将新闻和公告纳入系统。

这为 V2 的 Evidence Collector 提供了基础。

但 V1 中新闻更多用于情绪和热度判断，尚未完全转化为可追溯的结构化证据。

### 2.4 事件驱动雏形

V1 已经具备事件到商品、商品到股票的初步映射能力。

例如：

```text
六氟化钨紧缺
  ↓
钨 / 电子特气
  ↓
中船特气 / 中钨高新
```

这说明系统已经具备事件推理的雏形。

---

## 3. 当前主要问题

### 3.1 规则维护成本上升

随着事件类型增加，YAML 或硬编码规则会快速膨胀。

如果继续手工维护，规则会越来越难以审计和复用。

### 3.2 缺少统一知识结构

V1 中的映射大多是面向具体事件的规则。

但真实产业链关系是长期稳定的。

例如：

```text
六氟化钨 -> 电子特气 -> 半导体材料 -> 半导体制造
```

这条关系不应该只存在于某个事件规则中，而应该沉淀为长期知识。

### 3.3 缺少证据追踪

系统需要知道一条关系来自哪里。

例如：

```text
中船特气 produces 六氟化钨
```

这条关系可能来自：

- 年报。
- 招股说明书。
- 官网产品目录。
- 公告。
- 新闻。

如果没有证据追踪，关系的可信度无法评估。

### 3.4 缺少验证闭环

当前系统可以推理某个事件影响某些股票，但还需要验证：

```text
这些股票在事件发生后是否出现显著反应？
是否跑赢行业？
是否跑赢指数？
该事件类型的历史命中率如何？
```

没有验证闭环，系统无法区分有效关系和噪音关系。

### 3.5 可解释性不足

用户最终需要知道：

```text
为什么这只股票与事件相关？
它和事件之间的路径是什么？
这条路径有多少证据？
历史上类似事件是否有效？
```

如果系统只输出股票列表，用户很难信任结果。

---

## 4. 为什么不能继续堆规则

规则系统适合 MVP。

但当系统进入事件驱动阶段，规则会遇到扩展瓶颈：

1. 同一商品可能影响多个行业。
2. 同一公司可能涉及多个产品。
3. 同一事件可能通过多个路径影响股票。
4. 同一关系可能来自多个证据源。
5. 同一事件类型在不同市场环境下影响不同。

这些问题无法通过简单的字典映射长期解决。

---

## 5. V2 升级必要性

V2 的必要性来自三个方向：

### 5.1 产业关系需要长期沉淀

产业链关系不是一次性规则，而是可以不断积累的知识资产。

### 5.2 事件影响需要图谱推理

事件往往不是直接影响股票，而是通过商品、材料、行业、概念逐层传播。

### 5.3 推理结果需要市场验证

只有通过行情验证，系统才能知道哪些关系真的有效。

---

## 6. V1 可复用资产

V2 不需要重写 V1。

以下能力可以继续复用：

- 行情采集。
- 新闻采集。
- 公告采集。
- 调度系统。
- Web 基础框架。
- 股票详情页。
- 趋势评分。
- 事件 YAML 种子数据。

这些模块将作为 V2 的基础设施。

---

## 7. V1 到 V2 的核心转变

```text
V1：规则驱动
V2：知识驱动
```

```text
V1：新闻 -> 股票
V2：新闻 -> 事件 -> 图谱 -> 股票 -> 验证
```

```text
V1：输出结果
V2：输出推理路径、证据来源和历史验证
```

---

## 8. 结论

AStock V1 已经完成“能跑”的阶段。

V2 的重点不是继续增加更多指标，而是建立：

```text
Evidence Layer
Knowledge Graph
Reasoning Engine
Validation Loop
Explainability Layer
```

这五层将决定 AStock 是否能从普通量化看板升级为事件驱动分析平台。

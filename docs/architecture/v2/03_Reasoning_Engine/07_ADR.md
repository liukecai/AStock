# 07 ADR

> 本文档记录 AStock V2 Reasoning Engine 阶段的关键架构决策。

---

## ADR-001：推理必须基于结构化事件

### 状态

Accepted

### 背景

原始新闻文本存在歧义，不能直接进入图谱推理。

### 决策

所有事件必须先经过 Event Extraction 和 Event Normalization，形成结构化事件后再进入 Reasoning Engine。

### 影响

优点：

- 推理输入稳定。
- 便于去重。
- 便于验证。

代价：

- 需要维护事件类型和标准化逻辑。

---

## ADR-002：事件先映射到实体，再推理到股票

### 状态

Accepted

### 背景

直接从事件关键词映射到股票会导致规则膨胀和不可解释。

### 决策

采用：

```text
事件 -> 目标实体 -> 图谱路径 -> 股票
```

而不是：

```text
事件关键词 -> 股票列表
```

### 影响

优点：

- 路径可解释。
- 中间实体可复用。
- 支持多跳推理。

代价：

- 依赖图谱质量。
- 初期图谱不足时召回可能偏低。

---

## ADR-003：Exposure 使用连续分数

### 状态

Accepted

### 背景

股票与产业链节点之间不是简单相关或不相关。

### 决策

Exposure 使用 0.0 到 1.0 的连续分数。

### 影响

优点：

- 可排序。
- 可融合多因子。
- 可表达弱相关和强相关。

代价：

- 需要设计权重和阈值。

---

## ADR-004：Final Score 与 Confidence 分离

### 状态

Accepted

### 背景

一个股票可能因为事件和趋势得分较高，但证据置信度较低。

### 决策

分别保存：

```text
final_score
confidence
```

### 影响

优点：

- 避免高分低可信结果被误读。
- 前端可明确展示风险。

代价：

- 用户需要理解两个指标。

---

## ADR-005：长路径必须衰减

### 状态

Accepted

### 背景

图谱多跳查询容易产生过宽泛结果。

### 决策

路径分数引入 depth_decay。

推荐最大深度为 4。

### 影响

优点：

- 控制噪音。
- 提高解释质量。

代价：

- 可能遗漏远距离影响路径。

---

## ADR-006：历史验证用于校准交易有效性

### 状态

Accepted

### 背景

产业关系真实，不代表市场每次都会交易该关系。

### 决策

Validation 结果用于校准事件权重、路径权重和股票响应权重。

### 影响

优点：

- 系统可持续学习。
- 可识别噪音关系。

代价：

- 需要长期积累历史数据。

---

## ADR-007：推理结果必须保存路径

### 状态

Accepted

### 背景

如果只保存候选股票和分数，前端无法解释为什么股票相关。

### 决策

Reasoning Result 必须保存：

```text
reason_path
path_score
edge_details
evidence_ids
```

### 影响

优点：

- 支持 Why This Stock。
- 支持人工复核。
- 支持路径级验证。

代价：

- 存储结构更复杂。

---

## ADR-008：Reasoning Engine 不直接生成买卖建议

### 状态

Accepted

### 背景

推理结果只是事件影响分析，不等于交易建议。

### 决策

Reasoning Engine 输出候选影响、暴露度和解释，不输出买入、卖出、仓位建议。

### 影响

优点：

- 降低误导风险。
- 保持系统定位为研究分析工具。

代价：

- 用户需要结合自身判断使用结果。

---

## 结论

Reasoning Engine 的核心决策是：

```text
结构化事件输入
图谱路径推理
连续暴露度评分
分数与置信度分离
推理结果保存路径
市场验证校准权重
```

这些决策保证推理过程可解释、可复核、可验证。

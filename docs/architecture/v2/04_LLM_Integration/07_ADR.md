# 07 ADR

> 本文档记录 AStock V2 LLM Integration 阶段的关键架构决策。

---

## ADR-001：LLM 作为抽取器，不作为决策器

### 状态

Accepted

### 背景

LLM 擅长理解文本，但不适合作为不可审计的最终投资判断模块。

### 决策

LLM 只用于：

- 文本抽取。
- 事件识别。
- 候选实体生成。
- 候选关系生成。
- 解释文本辅助生成。

最终推理和评分由 Knowledge Graph、Reasoning Engine、Scoring Engine 和 Validation Engine 完成。

### 影响

优点：

- 降低幻觉风险。
- 保持系统可审计。
- 输出结果更稳定。

代价：

- 需要额外候选层和审核流程。

---

## ADR-002：LLM 输出必须进入候选层

### 状态

Accepted

### 背景

LLM 输出可能存在错误、过度推断或格式问题。

### 决策

LLM 输出不直接写入正式图谱，而是进入：

```text
candidate_entities
candidate_relations
candidate_events
```

通过自动校验或人工审核后，才进入正式图谱。

### 影响

优点：

- 防止正式图谱被污染。
- 支持审核和回滚。
- 支持质量控制。

代价：

- 增加数据流复杂度。

---

## ADR-003：所有 LLM 输出必须是 JSON

### 状态

Accepted

### 背景

自然语言输出难以稳定解析和入库。

### 决策

所有 LLM 抽取结果必须输出 JSON，并通过 Schema 校验。

### 影响

优点：

- 易于自动化处理。
- 易于校验。
- 易于失败重试。

代价：

- Prompt 设计更严格。
- 模型输出失败时需要重试或拒绝。

---

## ADR-004：evidence_text 为强制字段

### 状态

Accepted

### 背景

如果 LLM 只输出结论，不输出原文依据，无法审计其真实性。

### 决策

每条 entity、relation、event 输出必须包含 evidence_text。

缺少 evidence_text 的结果不得进入候选层。

### 影响

优点：

- 结果可追溯。
- 审核更高效。
- 降低幻觉风险。

代价：

- 输出字段更多。
- 长文本处理更复杂。

---

## ADR-005：LLM 不进入实时查询链路

### 状态

Accepted

### 背景

实时调用 LLM 会导致响应慢、成本高、结果不稳定。

### 决策

LLM 抽取作为离线任务执行。

在线 API 只读取已入库结果。

### 影响

优点：

- API 响应稳定。
- 成本可控。
- 结果可审计。

代价：

- 数据存在处理延迟。

---

## ADR-006：Prompt 必须版本化

### 状态

Accepted

### 背景

Prompt 变化会影响抽取结果。

### 决策

所有 Prompt 必须记录：

```text
prompt_name
prompt_version
model_name
```

抽取结果必须记录使用的 prompt_version。

### 影响

优点：

- 便于回溯。
- 便于比较不同 Prompt 效果。
- 便于问题排查。

代价：

- 需要维护 Prompt 版本管理。

---

## ADR-007：高质量来源可自动通过，低质量来源必须审核

### 状态

Accepted

### 背景

不同来源可信度差异很大。

### 决策

年报、招股书、公告、官网产品目录在高置信抽取时可自动通过。

新闻、RSS、互动易、招聘、社交讨论必须进入人工审核或多源确认。

### 影响

优点：

- 平衡效率和质量。
- 降低低质量来源污染图谱风险。

代价：

- 需要维护审核流程。

---

## ADR-008：解释生成只能基于已知路径和证据

### 状态

Accepted

### 背景

LLM 生成解释时可能自由发挥，添加不存在的关系。

### 决策

解释生成的输入必须包含：

```text
event
stock
reason_path
evidence_list
score_breakdown
```

LLM 只能基于这些输入生成说明。

### 影响

优点：

- 解释更可信。
- 避免新增幻觉关系。

代价：

- 解释生成依赖后端准备完整上下文。

---

## 结论

LLM Integration 阶段的核心决策是：

```text
LLM 只抽取
输出必须结构化
候选层隔离风险
证据文本强制绑定
离线执行
人工审核兜底
```

这些决策保证 AStock 可以使用 LLM 提高知识构建效率，同时不牺牲系统可信度。

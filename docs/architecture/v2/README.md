# AStock V2 Architecture

> A股产业知识图谱（Knowledge Graph）+ 事件推理引擎（Reasoning Engine）升级设计文档总览。
>
> 本目录用于指导 AStock 从 V1 的"行情 + 舆情 + 规则选股系统"，升级为 V2 的"产业知识图谱 + 事件推理 + 影响验证闭环系统"。

---

## 1. V2 升级目标

```text
事件输入 → 事件结构化 → 产业链知识图谱推理 → 股票暴露度计算
→ 趋势、舆情、事件多因子融合 → 收益验证与权重校准 → 可解释化 Web 展示
```

V2 不追求让大模型直接给出投资结论，而是让大模型参与信息抽取和关系补全，让知识图谱负责稳定推理，让行情数据负责验证关系是否真的有效。

---

## 2. 核心设计原则

| 原则 | 说明 |
|---|---|
| 规则库升级为确定性底座 | 现有 YAML 规则库保留，作为高置信度种子写入 Knowledge Graph |
| LLM 只做结构化抽取 | LLM 负责从年报、招股书、公告、新闻中抽取候选关系，输出必须经过校验后才能入正式图谱 |
| 图谱存储长期知识 | 解决"公司涉及什么产品""事件影响哪些商品""股票与事件之间的路径是什么" |
| 行情验证短期有效性 | 解决"这条关系过去是否带来超额收益""哪些关系只是概念噪音" |

## 4. V2 成功标准

V2 完成后，系统应能回答以下问题：

1. 某只股票为什么和某个事件相关？
2. 某个事件会影响哪些商品、材料、行业、股票？
3. 某条产业链关系来自哪里，置信度多少？
4. 同类事件过去是否真的带来超额收益？
5. 哪些关系是规则库确认的，哪些是 LLM 候选的，哪些经过市场验证？
6. 用户能否在 Web 页面看到完整推理路径？

---

## 5. 非目标

V2 暂不追求：

- 自动交易和实盘下单。
- 高频分钟级事件交易。
- 完全无人审核的 LLM 图谱写入。
- 对所有传闻直接生成买卖建议。

---

## 6. 总结

```text
LLM 负责理解和抽取
Knowledge Graph 负责记忆和推理
行情数据负责验证和校准
Web 前端负责解释和展示
```

这四层组合起来，才能真正实现：

> 供应链扰动 → 资源重估 → 板块定价重构 → 股票影响推理。

# 前端 UI 改造方案：决策信息链透明化

## 目标

当前前端已经可以展示信号列表和事件列表，但“为什么这个标的会排在这里”的解释层仍然不够透明。改造目标不是再堆更多数字，而是把现有后端已经产出的决策依据，组织成用户能顺着读下去的因果链：

```text
结论（是否值得研究）
<- 综合分怎么来的
<- 哪些新闻/事件在推动
<- 商品或舆情冲击如何传导
<- 具体证据和时间线是什么
```

本轮优先做前端信息架构和交互重组，尽量复用现有接口。若必须补接口，只允许做最小增量。

## 现状问题

### 1. 首页 `Dashboard`

文件：[frontend/src/views/Dashboard.vue](/Users/kecai/Documents/AStock/frontend/src/views/Dashboard.vue:1)

  - 这个分数主要由趋势推动，还是由新闻驱动；
  - 最近一条关键催化是什么；

- 但信号首页和个股页没有复用这套表达方式，系统体验割裂。
- 这页的 V2 证据组织方式，应该抽象成可复用组件，供个股详情页使用。

## 设计原则

### 1. 先结论，后证据

用户首先要知道“要不要继续研究”，其次才是“为什么”。页面应该先给出结论和排序，再允许逐层展开证据。

### 2. 解释优先于字段堆叠

不要新增一排排指标卡。要把指标变成句子、标签和路径：

- `趋势强` 不如 `均线多头 + 20日动量为正`
- `舆情 +0.63` 不如 `过去 24h 有 3 条偏正面催化，核心词为出口管制/涨价`

### 3. 同一套信息链模板复用到多个页面

首页、个股页、事件页应该共享一套“证据表达协议”，避免每页各说各话。

### 4. 默认收敛，按需展开

列表页只展示摘要；详情页展示完整链路；极细项通过折叠面板展开。

## 目标信息架构

### A. 首页改造成“信号 + 驱动摘要”双层列表

保留当前表格排序能力，但每行从单层表格升级成“两层卡片表格”：

#### 第一层：结果层

- 标的名 / 代码 / 板块
- 状态标签
- 综合分
- 研究权重
- 一个主导驱动标签：
  - `趋势主导`
  - `新闻主导`
  - `量能确认`

#### 第二层：解释层摘要

每行补一条可扫描摘要，格式建议：

```text
趋势 78 · 舆情 +0.42 · 量比 1.8x
最近驱动：价格上涨 / 供应偏紧 / 政策催化
最近 24h 提及 5 条，热度 Z 3.2
```

目的：

- 用户不必点进详情页就能判断“这是一只技术跟随股，还是事件催化股”。
- 排名前十的逻辑差异能一眼分开。

### B. 个股详情页新增“决策信息链总览”

当前页结构是“价格图 + 因子快照 + 新闻列表”。建议重组为四段：

#### 1. 决策总览 Hero

- 左侧：
  - 股票名 / 行业 / 代码
  - 当前状态
  - 一句决策摘要
- 右侧：
  - 综合分 Ring
  - 研究权重
  - 主导因子标签

一句决策摘要示例：

```text
趋势结构已成立，近期舆情偏正，热度抬升来自 2 条产业催化，当前更适合进入研究观察清单而非直接强化结论。
```

这句文案可以先按前端规则模板拼接，不要求后端生成自然语言。

#### 2. 决策链 Timeline

新增一个横向或纵向步骤条，明确展示：

1. `价格结构`
2. `情绪/新闻`
3. `热度/量能确认`
4. `综合评分结论`

每一步只放 1 到 2 个最关键证据。例如：

- `价格结构`：MA5 > MA20 > MA60，20 日动量 +8.3%
- `情绪/新闻`：近 50 条关联新闻均值 +0.21，核心词“涨价/出口/短缺”
- `热度/量能确认`：今日提及 6 条，热度 Z 3.1，量比 1.7x
- `综合结论`：总分 81，研究权重上限 6%

#### 3. 驱动拆解面板

把现在的 `因子快照` 改为三列或三块：

- 趋势驱动
- 舆情驱动
- 量能/热度驱动

每块统一结构：

- 当前分数
- 核心证据 2 到 3 条
- 行业偏差（已有字段可保留）

这样用户能明确知道：`总分高` 到底是被哪一条腿抬起来的。

#### 4. 证据流列表

把“近期信息流”升级为“证据流”，加入优先级分层：

- 关键驱动新闻
- 一般关联新闻

排序建议：

- 先按 `|sentiment|` 和时间排序
- 若未来接入事件反应分数，再优先显示命中事件链的新闻

每条证据卡至少展示：

- 时间
- 标题
- 来源
- 情绪方向
- 关键词
- 关联类型 `match_type`
- 置信度 `confidence`

### C. 事件页成为“可复用解释模板来源”

事件页不是本轮主改对象，但要作为样式和交互源头收敛：

- 保留“显式因果传导链”
- 将股票影响打分卡抽象成通用组件
- 后续个股页如命中商品事件，可直接复用 V2 传导卡片

## 文案与视觉方向

沿用现有深色基调，不推翻整体品牌，但要提升“研究台”而不是“普通后台”的感觉。

### 视觉要求

- 保留当前深色、荧绿高亮体系
- 强化层级边界：摘要、证据、结论三层必须视觉区分明显
- 尽量减少大表格感，增加研究卡片感
- 不引入 emoji 风格节点
- 不增加花哨动画，只保留轻量的展开/悬停反馈

### 文案要求

- 把指标翻译成研究语言
- 减少裸字段英文
- 明确区分：
  - `事实`
  - `推断`
  - `研究结论`

## 审核标准

我会按以下标准审核 antigravity 的实现：

### 1. 透明度

用户在首页不进入详情页，也能回答：

- 这只股票为什么上榜？
- 是趋势驱动还是新闻驱动？
- 研究权重为什么不是更高？

### 2. 连续性

个股详情页必须能顺着读出：

`证据 -> 因子 -> 评分 -> 结论`

而不是几个互不关联的模块拼盘。

### 3. 一致性

- 首页、个股页、事件页的标签体系一致
- 相同字段在不同页面的表达口径一致

### 4. 移动端可读性

- 不能再通过“隐藏解释字段”解决响应式
- 必须保留摘要层

### 5. 低侵入

- 尽量不改后端
- 不破坏现有分页、筛选、跳转
- 不改变 API 合约，除非是可选增强请求
